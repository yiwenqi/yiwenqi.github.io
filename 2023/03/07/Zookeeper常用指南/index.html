Zookeeper常用指南

ZK简介

Zookeeper是一个分布式的开源协调服务，可以用于在分布式系统中管理和协调大规模的计算机集群。它提供了一个简单的文件系统接口，可以用于协调分布式应用程序的各个组件，如配置管理、命名服务、集群管理等。

Zookeeper的主要特点是可靠性、高性能、可扩展性和简单性。它通过ZAB协议实现分布式一致性，可以保证在分布式系统中各个节点之间的数据一致性。Zookeeper还可以通过扩展机制来支持自定义协议和功能，使其具有更广泛的适用性。

zkCli

  1. ls：列出指定路径下的子节点列表，命令格式为ls path。
  2. create：创建一个新的znode节点，命令格式为create [-s] path data。其中，-s参数表示创建一个序列化的节点。
  3. delete：删除指定的znode节点，命令格式为delete path [version]。version表示指定删除哪个版本的节点，如果不指定则删除最新版本。
  4. get：获取指定节点的数据内容，命令格式为get path。
  5. set：设置指定节点的数据内容，命令格式为set path data [version]。其中，version表示指定修改哪个版本的节点，如果不指定则修改最新版本。
  6. stat：获取指定节点的元数据信息，命令格式为stat path。
  7. help：显示帮助信息，命令格式为help [cmd]。如果指定了cmd，则显示cmd命令的详细使用说明。
